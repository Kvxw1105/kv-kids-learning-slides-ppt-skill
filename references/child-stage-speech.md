# 儿童舞台演讲设计

## 一、产品定义

舞台演讲 PPT 是孩子的视觉背景和节奏伙伴。它服务观众理解、孩子记忆和现场情绪，不承担完整提词、课堂训练和教师讲解。

必须区分三个产物：

- `stage_deck.pptx`：正式放映，只有观众需要看的内容。
- `rehearsal_deck.pptx`：排练辅助，可显示关键词、动作图标和下一步提示。
- `speaker_script.md`：完整中英文讲稿、停顿、动作和发音提示。

正式版不得出现 `Say:`、`说：`、`Smile / Wave / Speak slowly`、教师提示、评分标准和完整段落讲稿。

## 二、四至六岁默认叙事弧

```text
打招呼
→ 展示一个具体的人或物
→ 做一个简单动作
→ 发生一个看得见的变化
→ 表达一种真实感受
→ 用一个可爱的愿望或动作结束
```

至少包含：一个具体细节、一个孩子能表演的动作、一个情绪变化。避免只有“它很好、我很喜欢、谢谢大家”的通用模板。


## 二点五、难度热插拔

年龄只决定可读性底线,不自动决定内容必须很少。演讲内容要按 `speech_profile` 热插拔。

```json
{
  "speech_profile": {
    "difficulty": "standard",
    "target_duration_seconds": 100,
    "target_english_words": [70, 110],
    "screen_density": "short_cues",
    "script_density": "moderate"
  }
}
```

- `light`: 第一次登台或英语基础弱,英文约 45-70 词。
- `standard`: 幼儿园正式展示,英文约 70-110 词。
- `rich`: 比赛或表达能力较强,英文约 110-180 词。

提高词汇量优先增加讲稿和备注,不要把正式屏幕塞满。正式屏幕仍保持大字、短句和干净留白。

## 三、屏幕规则

- 每页一个故事节点。
- 四至六岁每页最多两个主要文字块。
- 核心英文或中文通常 3–8 个词；辅助翻译不超过一行。
- 主标题 34–44pt，核心句 28–36pt，辅助翻译 22–28pt；任何观众可见文字不得低于 20pt。
- 观看距离默认 5 米。内容过多时拆页，不缩字号。
- 页面应由主图、原生教具或大字形成单一焦点，禁止普通办公白框连续出现三页。

## 四、讲稿规则

- 句子短，口语化，可呼吸。
- 英文单句通常 3–8 个词，复杂句拆开。
- 每 15–25 秒安排一个动作、声音、提问、惊喜或情绪变化。
- 不连续两页重复相同问候或结尾。
- 讲稿写在 `speaker` 或备注中，正式屏幕只保留关键词或短句。

## 五、图片与故事状态

视觉计划必须标明：`story_state`、`required_action`、`character_ids`、`must_show_change`。

叙事关键页的图片应让观众不读文字也能看出“正在发生什么”。文案说浇水，图片必须表现浇水或使用可编辑原生浇水图形；文案说花长大，应使用前后对比、成长序列或绽放状态。

同一主角的角色身份见 `references/character-continuity.md`。

## 六、舞台节奏

动画档位：`none`、`gentle`、`story-reveal`。

幼儿默认 `gentle`：只用淡入、出现和轻微上浮；每页一至三个节点。不能实现动画时，仍在 `stage.reveal_plan` 写明计划，并输出静态兼容版。

## 七、正式验收

- 观众看不到训练提示和完整讲稿。
- 屏幕、讲稿、双语顺序一致。
- 角色连续，图片动作对应台词。
- 字号适合投影。
- 至少使用三种同系统的页面骨架。
- 孩子只看缩略图也能回忆故事顺序。
