# 几何审查与视觉自审闭环

儿童 PPT 的发布检查分两层执行。结构正确不代表成品干净，视觉好看也不能替代几何安全。

## 第一层：代码与对象几何审查

在渲染前检查实际 PPTX，而不只检查 `slide_spec.json`：

- 文字框与分隔线、边框线、手绘下划线、装饰线不得相交。
- 核心标题、中文翻译和关键词标签周围必须保留安全间距。
- 文字不得压住人物脸部、花朵中心、太阳、云、花盆、浇水壶等故事主体。
- 标签不得超出页面，也不得与相邻标签过度重叠。
- 细线与文字的最小建议净距为 0.06 英寸；标题与装饰线建议 0.10 英寸以上。
- 任何自动修复都应修改源规格或生成代码后重新导出，禁止只在最终 PPTX 上做不可复现的手工补丁。

推荐命令：

```bash
python scripts/audit_pptx_geometry.py stage_deck.pptx --profile child_stage_speech --out geometry_report.json
```

若报告为 `BLOCKED`，先调整文字框高度、基线、容器内边距、分隔线位置或字号，再重新生成。

## 第二层：渲染后视觉审查

代码无法可靠判断字形实际渲染、字体替换、阴影、视觉重心和细微交叉，因此正式交付必须渲染复核。

```bash
python scripts/prepare_visual_review.py stage_deck.pptx --out-dir visual_review
```

该命令生成：

- 每页 PNG；
- `contact_sheet.png`；
- `visual_review.json` 模板；
- 当前 PPTX 的 SHA-256 绑定。

Agent 必须打开总览，并在必要时放大逐页图片。每页至少检查：

1. 文字与线框、气泡、边缘是否相交；
2. 中英文是否形成完整层级；
3. 字体替换后是否溢出、过小或显得像办公文档；
4. 主体、文字和装饰是否过密或过空；
5. 标签是否像临时散落，而不是完整组件；
6. 视觉焦点是否在三秒内明确；
7. 同一角色、色板和绘本语言是否连续。

把每页结果写入 `visual_review.json`：

- `PASS`：没有需要修改的问题；
- `WARN`：可交付但存在非阻断细节；
- `BLOCKED`：存在重叠、溢出、字体替换、结构失衡或明显毛坯感。

修复后必须重新渲染并重新审查，旧报告不得复用到新文件。

验证命令：

```bash
python scripts/validate_visual_review.py visual_review/visual_review.json --pptx stage_deck.pptx
```

## 发布要求

- 快速模式：几何审查必须通过；视觉审查可为警告。
- 智能配图模式：几何审查与视觉审查都必须完成。
- 重视觉工作室模式、比赛与正式舞台：两层审查均为发布阻断门。
- 若环境无法渲染，必须明确标记“视觉未验证”，不得声称已完成逐页视觉验收。
