# 路由与启动协议

## 1. 选择顶层路线

| 输入状态 | 路线 | 核心动作 |
|---|---|---|
| 主题、教材、教案、课文、图片或资料，目标是教学 | create-learning-deck | 重新设计教学主线与页面 |
| 幼儿园/小学演讲、自我介绍、双语展示、故事分享、比赛 | create-child-speech-deck | 设计舞台叙事、讲稿、动作节奏与正式/排练版本 |
| 用户指定本 Skill 模板或儿童模板 | fill-child-template | 保留模板语法，填充语义槽位 |
| 用户上传 PPTX 并要求沿用、重做或改编 | adapt-user-pptx | 先渲染和盘点原文件，再决定保留或重构 |
| 已有课件，要求补配图、讲稿、练习、答案、互动或修复 | enhance-learning-deck | 保持目标范围，定向增强 |

只选一条顶层路线。演讲路线不得复用课堂教学默认页型、训练提示卡和可见教师讲稿。

## 2. 场景判定优先于学科判定

用户说“不是上课”“上台讲”“双语演讲”“幼儿园展示”“比赛”“自我介绍”时，优先选择 `create-child-speech-deck`。主题即使是植物、动物或科学，也不能自动路由到课堂课件。

## 3. 信息不足的处理

已经知道主题、年龄和输出时直接执行。只有缺失信息会改变内容结构时，最多提出一个高杠杆问题。儿童演讲最有价值的问题通常是：

> 她想讲哪个真实的小东西、人物或经历？

用户授权“都你来定”时，加入一个具体物体细节、一个动作和一个情绪变化，并在 `course_brief.json` 记录假设。

## 4. 默认假设

### 课堂路线

- 比例：16:9。
- 场景：教师课堂讲授型。
- 视觉模式：smart。
- 小学课时：35–40 分钟。
- 文字和常规图形默认可编辑。

### 儿童演讲路线

- 比例：16:9。
- 年龄未给但出现“幼儿园”：按 5 岁、`4-6` 规划。
- 时长未给：按 60–90 秒规划。
- 视觉模式：smart。
- 双语未说明：采用 `paired_alternating`，每个故事节点先英文后中文；用户说英语演讲时改为 `english_primary`。
- 默认交付：正式放映版 + 演讲稿；排练版仅在用户需要或任务复杂时生成。
- 正式版屏幕角色：`visual_support`，不得显示完整讲稿和训练指令。

## 5. 输入盘点

通用字段：年龄/年级、主题、时长、场景、已有素材、视觉偏好、硬性内容、禁止项、输出格式。

儿童演讲追加：演讲者姓名或占位符、双语模式、真实故事细节、是否需要排练版、是否使用本人照片、角色服装与发型、舞台观看距离。

涉及教材版本、科学事实、历史人物或现实资料时先核验来源。闭卷素材任务只使用用户提供材料。
