# 响应式绘本排版引擎

## 目标

把“固定模板填内容”升级为“先测量内容，再选择构图，再做光学校正”。页面必须像儿童绘本的一页，而不是若干矩形卡片的排列。

## 四阶段

1. **测量**：统计屏幕文字量、语言数量、图像数量、关键词数量、页面角色与动作节点。
2. **选择**：从绘本构图原型中选择一个，不连续三页使用同一原型。
3. **缩放**：内容越少，主图与核心文字越大；内容越多，切换布局而不是无限缩字。
4. **光学校正**：允许主角、花朵、气泡等跨越数学网格 3%–8%，以获得自然的视觉重心。

## 内容量与默认尺度

| 内容量 | 屏幕文字单位 | 主视觉建议 | 文字建议 |
|---|---:|---:|---:|
| sparse | 0–10 | 42%–52% 页面面积 | 110%–125% 基准字号 |
| medium | 11–24 | 34%–44% | 100%–110% |
| rich | 25+ | 26%–36% | 90%–100%，必要时换布局 |

中文单字和英文单词都算一个内容单位。不要因为中文字符少就把中文做得极小。

## 绘本构图原型

- `full-bleed-caption-island`：封面或结束页；主图越界，标题落在独立岛屿。
- `character-speech-cloud`：人物与对话气泡；适合问候、自我介绍。
- `wraparound-object-story`：插画环绕文字；适合介绍物体和故事。
- `diagonal-action`：动作沿对角线推进；适合浇水、奔跑、操作。
- `weather-triptych`：三段场景；适合太阳、花、雨或并列比较。
- `three-vignette-growth`：三个小分镜；适合成长、前后变化。
- `emotion-closeup`：人物或物体特写；适合开心、惊喜和收束。
- `object-constellation`：物体居中，词汇围绕；词汇必须用花瓣、叶片或胶囊组装。
- `picturebook-window`：通用绘本窗口；边缘必须有框景或植物包边。

运行：

```bash
python scripts/plan_storybook_layout.py slide_spec.json --out slide_spec_planned.json --density balanced
```

## 禁止

- 不得连续三页使用同一左右分栏。
- 不得在内容很少时保持小主图、小字号和大空白。
- 不得把三个词直接散放在空白处；必须组成词汇组件家族。
- 不得把每页都做成圆角矩形卡片。
- 不得以“防重叠”为理由把所有对象推到彼此很远。
