# 儿童教学设计合约

## 教学闭环

默认结构：经验唤起 → 问题/冲突 → 观察或操作 → 表达猜想 → 规律揭示 → 立即练习 → 典型误区 → 迁移应用 → 离堂检测。

不是每节课都要机械出现九段，但不得缺少“理解形成”和“学习验收”。

## 学习目标

目标必须使用可观察行为：指出、分类、比较、解释、计算、复述、应用、创作。避免只写“了解、掌握、感受”。每个目标在 `lesson_map` 中关联：

- evidence_slides：在哪些页面看到孩子学会；
- misconceptions：最可能的错误；
- remediation：如何纠错；
- transfer：如何迁移到新情境。

## 页面双层结构

屏幕层：孩子要看的问题、图像、关键词、选项、算式、操作提示和反馈。

教师层：本页目标、讲法、停顿、预期回答、追问、错误回答、纠错、翻页条件和衔接语。优先放演讲者备注；不支持备注时输出 `teacher_notes.md`。

## 练习梯度

1. 识别：一眼判断、指认、配对。
2. 模仿：沿着示例完成一步。
3. 变式：改变表面情境或条件。
4. 解释：说出为什么。
5. 迁移：在新场景使用规律。

难度来自条件、表示和推理，不靠不必要的大数、长句或超纲知识。

## 互动

互动必须服务学习证据。支持举手、口头回答、摆一摆、画一画、圈一圈、连线、排序、投票、找错、预测和开放表达。PPT 动画只承担聚焦、分步揭示、状态变化和反馈，不做无意义飞入。
